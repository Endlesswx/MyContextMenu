# 右键菜单管家（ContextMenuManager）

一款针对 **Windows 10 / 11** 的系统原生右键菜单管理工具。**用完即走**，不常驻后台，直接修改系统注册表中的原生菜单，而不是另建一套自绘菜单。

绿色单文件，无需安装。主程序：`发布\右键菜单管家.exe`（约 70 KB）。

## 一、能管理什么

| 分类 | 管理对象 | 对应注册表位置 |
|------|----------|----------------|
| 桌面右键菜单 | 桌面 / 文件夹窗口空白处的右键菜单 | `HKCR\Directory\Background`、`HKCR\DesktopBackground` |
| 「新建」菜单 | 右键 → 新建 子菜单中的各文件类型 | 各 `.ext\ShellNew`；商店应用（MSIX）内置项另行识别 |
| 文件右键菜单 | 右键任意文件时的菜单（所有类型通用） | `HKCR\*`、`HKCR\AllFilesystemObjects` |
| 文件夹右键菜单 | 右键文件夹时的菜单 | `HKCR\Directory`、`HKCR\Folder` |
| 驱动器右键菜单 | 右键磁盘（C:、D: 等）时的菜单 | `HKCR\Drive` |
| 「发送到」菜单 | 右键 → 发送到 子菜单中的项目 | `%APPDATA%\Microsoft\Windows\SendTo` |
| 按后缀名查询 | 指定后缀名（如 `.txt`、`.jpg`）文件独有的菜单 | `.ext`、关联 ProgID、`SystemFileAssociations` |

每个分类都同时覆盖两类菜单来源：
- **静态菜单项（verb）**：注册表 `shell\` 下的项，如「用 VS Code 打开」。
- **Shell 扩展（COM 处理程序）**：`shellex\ContextMenuHandlers\` 下由 DLL 提供的项，如压缩软件、网盘、共享等。

## 二、能做什么操作

- **屏蔽 / 恢复**：全部采用**可逆**方式，随时可一键还原（原理见下）。
- **删除**：从注册表中移除。删除前会**自动导出 `.reg` 备份**（「发送到」项则移入备份文件夹），双击备份即可完整还原。
- **新增**：
  - 新增普通右键菜单项（可指定显示位置、程序、命令、图标、是否仅 Shift 显示、是否置顶）；
  - 为指定后缀名新增「新建」菜单项；
  - 新增「发送到」项（选择目标程序或文件夹）。
- **定位**：一键在注册表编辑器 / 资源管理器中定位到该项。
- **搜索**：按名称 / 命令 / 路径实时过滤。
- **排序**：点击列表的列标题（状态 / 名称 / 类型 / 位置 / 命令）即可升/降序排序，标题上的 ▲▼ 指示当前排序。
- **Win11 经典菜单**：一键在 Win11 精简菜单与经典完整菜单之间切换（仅当前用户）。
- **重启资源管理器**：部分修改需重启 explorer 生效，提供一键重启。

## 三、屏蔽的实现原理（均可逆、非破坏性）

| 类型 | 屏蔽方式 | 恢复方式 |
|------|----------|----------|
| 静态菜单项 verb | 写入 `LegacyDisable` 空值 | 删除该值 |
| Shell 扩展（CLSID 在默认值） | 默认值 CLSID 前加 `-`，使其无法解析 | 去掉 `-` |
| Shell 扩展（CLSID 在键名） | 键名前加 `-` 重命名 | 去掉 `-` 重命名回 |
| 「新建」模板 | 将 `ShellNew` 键重命名为 `ShellNew-` | 重命名回 `ShellNew` |
| 「新建」·商店应用（内置） | 改写 MrtCache 中缓存的显示名，追加超长内容使其从菜单消失 | 去掉追加内容，还原原始显示名 |
| 「发送到」项 | 给文件设置「隐藏」属性 | 取消隐藏属性 |

这些都是业界成熟做法（如 NirSoft ShellExView 屏蔽扩展、ShellNew 重命名法），不删除任何数据，因此**可完全还原**。

## 四、使用说明

1. 双击 `右键菜单管家.exe`。
2. 程序会询问是否以管理员身份运行。
   - 系统级菜单位于 `HKLM`（`HKEY_LOCAL_MACHINE`），**修改它们需要管理员权限**。
   - 选择「是」→ 通过 UAC 提权后重启；选择「否」→ 以普通权限运行，可浏览全部菜单，但对系统级项目的修改会提示权限不足。
   - 也可在主界面左上角点击「未以管理员运行（点此提权）」随时提权。
3. 左侧选择分类，右侧列表中对每一项进行 屏蔽 / 恢复 / 删除 / 定位。
4. 删除的备份存放在程序目录下的 `backups\` 文件夹（状态栏「打开备份文件夹」可直达）。

> 提示：多数修改会立即写入注册表并通知系统刷新；个别项（尤其 Win11 菜单风格切换）需要**重启资源管理器**后才在右键菜单中体现，可用顶栏按钮一键重启。

## 五、技术栈

- **C# + WPF**（.NET Framework 4.0 目标框架，Win10/Win11 均自带运行时，**无需额外安装 .NET**）。
- 单一可执行文件，无外部依赖。
- 通过 `Microsoft.Win32.Registry` 直接读写注册表；`SHLoadIndirectString` 解析菜单的本地化资源名；`SHChangeNotify` 通知外壳刷新。

## 六、命令行参数（供进阶 / 自检使用）

| 参数 | 说明 |
|------|------|
| `--noelevate` | 跳过管理员提权询问，直接以当前权限启动 |
| `--selftest [文件]` | 运行内置自检（仅在 `HKCU` 测试键与临时目录内操作，不触碰真实菜单），输出结果并以退出码 0/2 表示通过/失败 |
| `--dump [文件]` | 扫描当前系统的全部菜单并导出为文本，便于排查 |

## 七、从源码构建

需要能定位到 MSBuild（随 Visual Studio 或 Build Tools 安装）。

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File build.ps1
```

脚本会编译 `src\ContextMenuManager.csproj`（Release），并将产物复制为 `发布\右键菜单管家.exe`。

## 八、注意事项

- **商店应用（内置）的「新建」项**：Windows 11 的部分「新建」菜单项（如「文本文档」来自记事本、「BMP 图像」来自画图）由**商店应用（MSIX）包清单**注册，而非传统 `ShellNew` 注册表键。本工具会识别并列出它们（位置显示「商店应用（内置）」），并支持**屏蔽 / 恢复**：屏蔽通过改写 `HKCU\...\Local Settings\MrtCache` 中缓存的显示名（追加超长内容使该项从菜单消失）实现，可随时“恢复”还原为原始名。此类项不支持「删除」（请用「屏蔽」代替）。屏蔽后通常需**重启资源管理器**才在右键菜单中体现。
- 修改系统菜单有一定风险，程序已尽量采用可逆方式并在删除前备份，但仍建议重要操作前自行留存系统还原点。
- 部分第三方软件会在启动 / 更新时重新写入自己的菜单项，被屏蔽的项可能因此重新出现，届时再次屏蔽即可。
- 本工具修改的是当前机器的注册表，不联网、不常驻、不收集任何信息。

## 九、目录结构

```
MyContextMenu\
├─ 发布\右键菜单管家.exe      ← 最终可执行文件（发给用户的就是这一个）
├─ build.ps1                  ← 构建脚本
├─ README.md
├─ 用户需求.txt
└─ src\
   ├─ ContextMenuManager.csproj
   ├─ app.manifest            ← asInvoker + Win10/11 兼容声明
   ├─ App.xaml(.cs)           ← 启动、提权、命令行入口、全局样式
   ├─ MainWindow.xaml(.cs)    ← 主界面
   ├─ AddItemWindow.xaml(.cs) ← 新增菜单项对话框
   ├─ Models.cs               ← 数据模型
   ├─ NativeMethods.cs        ← Win32 互操作
   ├─ RegistryHelper.cs       ← 注册表读写 / .reg 导出 / 重命名 / 删除
   └─ MenuService.cs          ← 扫描 / 屏蔽 / 新增 / 删除 / 自检核心
```
