# 右键菜单管家（ContextMenuManager）

管理 **Windows 10 / 11** 系统右键菜单的工具。用完就关，不驻留后台，直接改注册表里的原生菜单，不搞自绘那套。

绿色单文件，免安装。主程序：`发布\右键菜单管家.exe`（约 70 KB）。

## 一、能管理什么

| 分类 | 管理对象 | 对应注册表位置 |
|------|----------|----------------|
| 桌面右键菜单 | 桌面 / 文件夹窗口空白处的右键菜单 | `HKCR\Directory\Background`、`HKCR\DesktopBackground` |
| 「新建」菜单 | 右键 → 新建 子菜单中的各文件类型 | 各 `.ext\ShellNew`；商店应用（MSIX）内置项另行识别 |
| 文件右键菜单 | 右键任意文件时的菜单（所有类型通用） | `HKCR\*`、`HKCR\AllFilesystemObjects` |
| 文件夹右键菜单 | 右键文件夹时的菜单 | `HKCR\Directory`、`HKCR\Folder` |
| 驱动器右键菜单 | 右键磁盘（C:、D: 等）时的菜单 | `HKCR\Drive` |
| 「发送到」菜单 | 右键 → 发送到 子菜单中的项目 | `%APPDATA%\Microsoft\Windows\SendTo` |
| 按后缀名查询 | 指定后缀名（如 `.txt`、`.jpg`）文件独有的菜单 | `.ext`、关联 ProgID、`SystemFileAssociations` |

每个分类覆盖两类菜单来源：
- **静态菜单项（verb）**：注册表 `shell\` 下的项，比如「用 VS Code 打开」。
- **Shell 扩展（COM 处理程序）**：`shellex\ContextMenuHandlers\` 下由 DLL 提供的项，比如压缩软件、网盘的菜单。

## 二、能做什么操作

- **屏蔽 / 恢复**：全是可逆操作，随时能一键还原（原理见下）。
- **删除**：从注册表移除。删除前会自动导出 `.reg` 备份（「发送到」项移入备份文件夹），双击备份就能还原。
- **新增**：
  - 新增普通右键菜单项（指定显示位置、程序、命令、图标、是否仅 Shift 显示、是否置顶）；
  - 为指定后缀名新增「新建」菜单项；
  - 新增「发送到」项（选择目标程序或文件夹）。
- **定位**：一键在注册表编辑器或资源管理器中定位到该项。
- **全局搜索**：输入关键字自动跨所有分类搜索，可按名称、命令、路径、关联软件等字段过滤。加了防抖，不会卡。
- **排序**：点列表的列标题（状态 / 名称 / 类型 / 位置 / 命令）就能升降序排序，标题上的 ▲▼ 显示当前排序。
- **Win11 经典菜单**：一键在 Win11 精简菜单和经典完整菜单之间切换（仅当前用户）。
- **重启资源管理器**：有些修改需要重启 explorer 才生效，提供一键重启。

## 三、屏蔽的实现原理（均可逆、非破坏性）

| 类型 | 屏蔽方式 | 恢复方式 |
|------|----------|----------|
| 静态菜单项 verb | 写入 `LegacyDisable` 空值 | 删除该值 |
| Shell 扩展（CLSID 在默认值） | 默认值 CLSID 前加 `-`，使其无法解析 | 去掉 `-` |
| Shell 扩展（CLSID 在键名） | 键名前加 `-` 重命名 | 去掉 `-` 重命名回 |
| 「新建」模板 | 将 `ShellNew` 键重命名为 `ShellNew-` | 重命名回 `ShellNew` |
| 「新建」·商店应用（内置） | 改写 MrtCache 中缓存的显示名，追加超长内容使其从菜单消失 | 去掉追加内容，还原原始显示名 |
| 「发送到」项 | 给文件设置「隐藏」属性 | 取消隐藏属性 |

这些都是业界成熟方法（比如 NirSoft ShellExView 屏蔽扩展用的也是这套），不删数据，所以能完全还原。

## 四、使用说明

1. 双击 `右键菜单管家.exe`。
2. 程序会问要不要以管理员身份运行。
   - 系统级菜单在 `HKLM`（`HKEY_LOCAL_MACHINE`），改它们需要管理员权限。
   - 选「是」→ 通过 UAC 提权后重启；选「否」→ 普通权限运行，能看所有菜单，但改系统级项目会提示权限不够。
   - 也可以在主界面左上角点「未以管理员运行（点此提权）」随时提权。
3. 左侧选分类，右侧列表里对每项进行 屏蔽 / 恢复 / 删除 / 定位。
4. 删除的备份在程序目录下的 `backups\` 文件夹（状态栏点「打开备份文件夹」直达）。

> 提示：多数修改会立即写入注册表并通知系统刷新；个别项（特别是 Win11 菜单风格切换）需要重启资源管理器才能在右键菜单中看到效果，顶栏按钮可一键重启。

## 五、技术栈

- **C# + WPF**（.NET Framework 4.0，Win10/Win11 自带运行时，不用装 .NET）。
- 单一可执行文件，无外部依赖。
- 通过 `Microsoft.Win32.Registry` 读写注册表；`SHLoadIndirectString` 解析菜单的本地化资源名；`SHChangeNotify` 通知外壳刷新。

## 六、命令行参数

| 参数 | 说明 |
|------|------|
| `--noelevate` | 跳过管理员提权询问，直接以当前权限启动 |
| `--selftest [文件]` | 运行内置自检（只在 `HKCU` 测试键和临时目录操作，不碰真实菜单），输出结果，退出码 0 表示通过、2 表示失败 |
| `--dump [文件]` | 扫描当前系统的全部菜单并导出为文本，方便排查问题 |

## 七、从源码构建

需要能定位到 MSBuild（随 Visual Studio 或 Build Tools 安装）。

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File build.ps1
```

脚本会编译 `src\ContextMenuManager.csproj`（Release），并将产物复制为 `发布\右键菜单管家.exe`。

## 八、注意事项

- **商店应用（内置）的「新建」项**：Windows 11 有些「新建」菜单项（比如记事本的「文本文档」、画图的「BMP 图像」）是商店应用（MSIX）包清单注册的，不在传统 `ShellNew` 注册表键里。本工具能识别并列出它们（位置显示「商店应用（内置）」），支持屏蔽和恢复：屏蔽是通过改写 `HKCU\...\Local Settings\MrtCache` 里缓存的显示名（追加超长内容让这项从菜单消失）实现的，随时能恢复还原。这类项不支持删除，用屏蔽代替。屏蔽后通常要重启资源管理器才能在右键菜单看到效果。
- 改系统菜单有风险，程序尽量用可逆方式、删除前会备份，但还是建议重要操作前自己做个系统还原点。
- 有些第三方软件启动或更新时会重新写入自己的菜单项，被屏蔽的项可能又冒出来，再屏蔽一次就行。
- 本工具只改本机注册表，不联网、不驻留、不收集信息。

## 九、目录结构

```
MyContextMenu\
├─ 发布\右键菜单管家.exe      ← 最终可执行文件（发给用户的就是这一个）
├─ build.ps1                  ← 构建脚本
├─ README.md
├─ 用户需求.txt
└─ src\
   ├─ ContextMenuManager.csproj
   ├─ app.manifest            ← asInvoker + Win10/11 兼容声明
   ├─ App.xaml(.cs)           ← 启动、提权、命令行入口、全局样式
   ├─ MainWindow.xaml(.cs)    ← 主界面
   ├─ AddItemWindow.xaml(.cs) ← 新增菜单项对话框
   ├─ Models.cs               ← 数据模型
   ├─ NativeMethods.cs        ← Win32 互操作
   ├─ RegistryHelper.cs       ← 注册表读写 / .reg 导出 / 重命名 / 删除
   └─ MenuService.cs          ← 扫描 / 屏蔽 / 新增 / 删除 / 自检核心
```
